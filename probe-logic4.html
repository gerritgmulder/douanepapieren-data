Eenmalige probe

Deze pagina haalt de meest recente order op uit Logic4 inclusief regels en het eerste gerelateerde product. Het doel: zien welke velden Fonteyn invult — verkoper, kostprijs, artikelgroep — zodat we weten of we een real-time bruto-marge dashboard kunnen bouwen.

Order-statussen inventariseren

Probeert eerst een aantal officiële Logic4-endpoints; valt anders terug op een empirische sample (eerste 2000 orders → unieke statussen verzamelen). Doel: een complete lijst van status-IDs zodat we de Orderstatus-tool kunnen voeden met álle relevante bron-statussen (incl. "Wachten op minimaal 30% aanbetaling").

Bol-memoriaal API-scope (read-only probe)

Test of de Logic4 API memoriaal/dagboek-boekingen ondersteunt voor jullie installatie. Veilig: alleen GET/list-endpoints — geen boekingen worden aangemaakt of gewijzigd. Als één endpoint werkt = de Bookings-module zit in jullie scope = waarschijnlijk kunnen we automatisch memoriaal aanmaken. Als alles 404 geeft (zoals Debtors) = geen Logic4-write mogelijk, dan bouwen we een import-Excel-route.

Bol-memoriaal Route A — Payment/Order-write probe

Test of we via de Orders-module (die we wél hebben) betalingen kunnen koppelen aan orders. Werkbaar: 90% van Osman's werk zit in credit-regels = "klant X heeft order Y betaald", en dat is functioneel een betaling toevoegen aan een order. Probe gebruikt OrderId -1 (= bestaat zeker niet) zodat niets per ongeluk wordt aangemaakt. We zien per endpoint of het bestaat (Logic4 400 = "order -1 not found" → endpoint reageert) of niet (404 → buiten scope).
